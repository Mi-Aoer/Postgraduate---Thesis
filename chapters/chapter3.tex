\chapter{面向新闻领域知识图谱构建的高质量可控知识抽取框架}
\label{chap:framework}

\section{任务目标与问题定义}

\subsection{任务场景与研究目标}

本研究面向新闻领域知识图谱构建场景下的知识抽取任务。和通用文本处理任务不同的是，新闻领域文本往往来源广、更新快、表达方式不统一，而且同一事实可能会出现在不同平台、不同文体和不同叙述层级中，知识也通常以分散、隐含的方式存在于原始文本里，难以直接用于后续组织和利用。对于知识图谱构建来说，真正需要的是能够被进一步表示和存储的结构化知识。本文恰好关注的就是如何从多源异构的领域新闻文本中，稳定抽取出能够服务知识图谱构建的实体关系知识，并为后续的融合、入库和应用提供基础。这个任务场景对知识抽取提出了比普通信息抽取更高的要求，一方面来说，新闻文本中包含大量专业术语、事件信息和上下文依赖，单纯依赖人工规则或传统监督方法，往往面临标注成本高、迁移能力弱、难以适应领域表达持续变化等问题。另一方面，知识图谱构建并不只关心抽取得对不对，还关心抽取结果能不能用。如果输出结果存在实体边界不清、关系类型不匹配、结构不完整或格式不一致等问题，那么即便局部语义看上去合理，也仍然难以直接支撑图谱的稳定入库和持续更新。此外，考虑到实际场景中的本地部署、推理成本和长期运行需求，仅仅依赖超大模型进行在线抽取也并不现实。因此本文所讨论的任务本质上是一个同时受到数据质量、结果可控性和部署可行性约束的结构化知识获取问题。

本文的总体研究目标是：围绕领域知识图谱构建需求，提出一套兼顾高质量、可控性与落地性的知识抽取框架，使原始新闻文本能够较为稳定地转化为可组织、可入库、可服务下游应用的结构化知识。具体而言，本文希望实现三个层面的目标：第一，在训练数据层面，面向领域标注资源稀缺的问题，构建高置信度的银标数据，为后续模型训练提供更可靠的数据基础；第二，在抽取模型层面，面向资源受限和本地部署需求，构建轻量化的领域知识抽取方法，并通过约束机制提升输出结果的结构合法性和稳定性；第三，在应用层面，将抽取得到的结果进一步用于知识图谱构建与基础问答验证，检验所提出方法能否贯通从原始文本到结构化知识组织再到简单知识服务的完整流程。通过这样的设计，本文希望进一步回答“抽取结果能否真正进入图谱、支撑更新并服务应用”这一更接近实际场景的核心问题。

\subsection{核心问题分析}

结合前文任务场景分析，问题主要体现在以下三个方面。

\begin{enumerate}
    \item \textbf{高质量训练数据获取困难，直接制约后续模型学习效果}

    新闻领域文本中，同一类事实往往会以不同说法出现在不同文本里。表面上看语料很多，但真正能够直接用于模型训练的高质量标注数据并不充足。如果完全依赖人工标注，不仅成本高、周期长，一致性也难以长期保证；但如果直接使用自动生成结果，又容易把噪声和幻觉样本带入训练过程。这样一来，后续模型即使完成了领域适配，也可能学到的是不稳定甚至有偏差的抽取模式。因此如何以可接受的成本构建可靠的训练数据，是整个技术路线首先要解决的问题。

    \item \textbf{抽取结果难以同时兼顾准确性、可控性与部署可行性}

    大语言模型在复杂文本理解方面确实带来了新的可能，但在知识图谱构建场景中，能生成结果并不等于结果可以直接使用。一方面，直接依赖大模型长期在线抽取，会受到推理成本、响应时延和本地部署条件的限制；另一方面，即使采用小参数模型并进行参数高效微调，也不意味着输出天然就是规范和可控的。对于知识图谱来说，结果不仅要语义上大体正确，还要满足实体类型与关系类型的匹配要求，尽量避免出现类型冲突、关系不合理、结构难解析等问题。换句话说，这里的难点已经不只是模型精度本身，而是如何让抽取结果从看起来合理变成结构上合法以及工程上可用。

    \item \textbf{抽取方法不能停留在离线指标上，还必须服务下游图谱构建与应用验证}

    本文研究的目标是要把抽取结果真正用于知识图谱构建，前端文本经过抽取之后，结果还要能够进入后续的 schema 维护、知识组织、图谱存储和查询流程。如果抽取得到的三元组格式不稳定、语义冲突多、后处理成本高，那么即便句子层面的指标尚可，也很难支撑后续图谱入库与应用服务。因此，本文需要把“是否便于稳定入库”“是否能够支持基础检索与简单问答”纳入问题分析之中，从更完整的流程角度来审视知识抽取方法的有效性。
\end{enumerate}

综合来看，面向领域知识图谱构建的知识抽取是一个贯穿训练数据构建、可控抽取和下游落地的系统性问题。也正因为如此，本文后续将分别从高质量银标数据构建、轻量化本体约束抽取以及知识图谱构建与应用验证三个层面展开设计，以形成一条更完整、更可落地的技术路径。

\subsection{下游应用需求}

抽取结果最终是否有价值，不能只看句子层面的准确率，还要看它能否顺利进入图谱，能否被后续系统使用。

从知识图谱构建的角度看，下游最直接的需求是抽取结果能够被稳定组织和持续入库。如果抽取结果在实体边界、关系类型或输出结构上缺乏一致性，那么后续的 schema 维护、知识组织、图数据库存储和检索都会受到影响。因此，抽取得到的实体、关系及属性信息应当能够较自然地映射到schema之中，支持后续的图谱构建、知识存储与基本检索，并为后续增量更新预留空间。只有这样，知识抽取才能真正成为图谱构建流程中的稳定入口。

从应用验证的角度看，系统至少应能够支持两类较为基本但具有代表性的应用需求：一类是围绕单个实体展开的查询，如返回实体的属性信息、关联对象或相关描述；另一类是面向实体之间关系事实的查询，即根据给定实体对或关系约束返回图谱中已经组织好的知识结果。重点在于验证“文本输入—知识抽取—图谱构建—基础服务”这条链路的畅通。

因此，本文所面对的下游应用需求，本质上可以概括为两个方面：一是支撑知识图谱的稳定构建和存储；二是支撑实体关系查询和简单问答的基础知识服务。也正因为存在这样的下游约束，本文后续的框架设计必须同时考虑结果的结构合法性、一致性以及工程可落地性，使抽取能力真正服务于领域知识图谱的构建与应用验证。

其中，数据预处理处在整条链路的前端。新闻文本的表达方式不统一，原始语料里往往还混有重复信息、噪声内容和不规范表达。预处理的作用，就是先把输入整理到一个相对稳定的状态，作为后续自动标注和模型训练的基础。

Teacher 端和 Student 端在框架中的职责并不相同。Teacher 端更偏向于离线阶段的数据供给，重点是尽量构建出质量更高、噪声更低的训练样本；Student 端更偏向实际抽取阶段，重点是把这些训练数据转化为可部署、可持续运行的抽取能力。两者前后衔接，分别对应“先把数据做好”和“再把抽取做好”这两个层面。

图中上方的新闻领域本体 Schema 约束，贯穿了 Teacher 和 Student 两个环节。对于 Teacher 端，它限定了银标构建时的实体类型、关系类型和输出边界；对于 Student 端，它进一步约束抽取结果的结构合法性，减少类型不匹配和关系不合理的三元组。这样，前面的数据构建和后面的结果生成就可以保持在同一套领域语义框架下。

Student 端的输出还要继续进入图谱构建与基础服务验证环节。对本文来说，抽取得到三元组只是中间结果，更重要的是这些结果能否稳定入库，能否支持后续查询与简单问答。因此，图谱构建与验证放在链路末端可以用来检验前面各环节的整体可用性。后续如果在这个阶段暴露出来了问题，也可以反过来帮助定位哪一步还需要进一步优化。

\section{新闻领域本体模型设计}

本体（Ontology）是知识图谱构建的基础，它用于定义领域中的实体类型、关系类型和属性类型，并刻画它们之间的层次结构与语义关联。对于新闻领域知识图谱而言，本体模型不仅决定了图谱中“有哪些对象”“对象之间存在哪些联系”以及“对象具备哪些属性”，也为新闻文本中的知识抽取、结果组织和后续入库提供统一的模式约束。

本文以军事新闻为重点实例场景，对新闻领域本体模型进行场景化细化。本小节对实体类型、关系类型和属性类型进行了统一设计，下文将从本体结构、新闻领域实体类型、新闻领域关系类型以及属性类型与约束作用四个方面进行说明。

\subsection{本体结构}

本文构建的新闻领域本体模型采用“模式层—实例层”的两层结构。模式层用于定义领域中的实体类型、关系类型和属性类型，是知识图谱构建与知识抽取的统一语义基础；实例层则承载从具体新闻文本中抽取得到的实体实例、关系实例及其属性值。模式层回答的是“应当如何组织知识”的问题，实例层回答的是“从文本中实际获得了哪些知识”的问题，二者相互衔接，共同构成新闻领域知识图谱的基本表示框架。

考虑到军事新闻文本大多围绕某一事件展开，本文在本体结构设计中以“军事新闻事件”为核心，将行动主体、军事资产、地点、时间、数量、任务内容、事件结果和资源等要素组织到统一的事件语义框架之中。其中，军事新闻事件实体用于描述新闻报道中的核心事件；行动主体、军事资产等实体用于表示事件的主要参与方及关键对象；地点、时间和数量等实体用于补充事件发生的时空背景和规模信息；任务内容、事件结果和资源等实体则用于进一步刻画事件的目的、后果与支撑条件。通过这样的结构设计，可以将原本分散在新闻文本中的事件信息及其上下文要素进行统一表达，使知识图谱能够较完整地反映军事新闻中的事件逻辑和语义关联。

\subsection{新闻领域实体类型}

在军事新闻知识图谱中，实体类型是知识组织的基本单元。结合军事新闻文本的内容特点，本文围绕事件表达需求设计了若干核心实体类型，如表~\ref{tab:military_news_entity_types}示。整体上看，这些实体可以划分为三类：一类是事件核心类实体，即军事新闻事件；一类是事件参与类实体，包括行动主体和军事资产等，用于表示事件中的主要参与者和关键对象；另一类是事件上下文类实体，包括地点、时间、数量、任务内容、事件结果和资源等，用于补充描述事件发生的背景、过程和结果。

\section{方法与应用衔接思路}

\subsection{训练数据构建思路}

在新闻领域知识图谱构建任务里，训练数据的质量会直接影响后续抽取模型的效果，因为领域文本一般来说专业性比较强，而高质量人工标注又成本较高，所以本文不依赖大规模的人工标注，而采用基于大语言模型的数据集构建思路，先解决训练样本的质量问题。

具体来说，需要先对原始语料进行清洗、规范化和分句处理，并从中选取少量种子样本开展初始抽取；再结合错误反馈对提示进行 SO-CoT 自优化，使模型逐步形成更适合当前任务的抽取方式；在此基础上，对全量语料进行实体关系联合抽取，并引入多模型事实评估，从因果逻辑、语义完整性和领域常识等方面对结果进行筛选；对未通过评估的结果再回流重抽，最终形成质量更高的一批新闻领域银标数据，这样就可以得到可用于后面模型训练的高质量数据集了。

\subsection{可控抽取实现思路}

可控抽取的重点是在便于本地部署的前提下，让抽取结果尽可能稳定且可用。本文的可控抽取思路是基于可以在有限资源的机器上部署的基础上，进一步提高输出结果的合法性与稳定性。具体做法是：先用目前主流的小参数开源指令模型作为基座，利用 QLoRA 微调方法，和上一步生成的数据集样本来让基座模型完成新闻领域的适配；再在推理阶段引入新闻领域的实体关系约束，过滤掉不合理的组合；最后通过多候选结果生成与选择机制，降低单次生成带来的波动，输出更稳定的结构化结果。这样，Student 端就实现了一个兼顾低资源部署成本和高质量抽取效果的抽取模块。

\subsection{图谱构建与验证思路}

在获得结构化抽取结果后，就可以进一步考虑这些结果如何真正用于知识图谱构建。这一步需要将抽取得到的实体、关系和属性信息进一步整理并映射到 schema 中，形成可统一组织的知识单元。在此基础上结合图数据库完成知识存储与基本检索，使新闻知识抽取结果转化为图谱层面的可管理知识。

在完成图谱构建后，本文通过基础查询和简单问答对结果进行验证，主要考察图谱是否能够支持实体查询、关系查询等基本需求，以及是否能够完成简单问答的功能。通过这种方式，可以更直观地检验前述抽取结果是否真正具备可用性。

\section{本章小结}

本章面向新闻领域知识图谱构建需求，对高质量可控知识抽取任务的目标、问题和整体框架进行了统一说明，并对军事新闻领域本体模型的设计进行了详细阐述。围绕训练数据质量、抽取结果可控性以及下游图谱应用需求，本章建立了由银标数据集构建、可控知识抽取和图谱应用验证组成的整体技术路线，为后续具体方法展开提供了基础。
