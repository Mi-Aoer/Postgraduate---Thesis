\chapter{面向轻量化本地部署的参数高效本体约束知识抽取方法}
\label{chap:lightweight-ke}

\section{问题描述}

\section{参数高效的本体约束知识抽取流程}

参数高效的本体约束知识抽取方法是一种结合大语言模型领域适配能力与领域本体约束机制的知识抽取方法。这种方法首先利用参数高效微调技术，使基座模型具备对领域文本的理解与实体关系联合抽取能力；随后，在模型生成候选结果的基础上，结合领域本体规则对抽取结果进行约束、筛选与重排序，从而输出更加稳定、更加符合领域结构要求的知识结果。

  \caption{参数高效的本体约束知识抽取流程}
  \label{fig:efficient_ontology_constrained_extraction_process}
\end{figure}

图~\ref{fig:efficient_ontology_constrained_extraction_process}展示了参数高效本体约束抽取的整体流程。该流程主要分为四个阶段：离线领域适配、候选结果生成、本体约束与候选评分、结构化输出。以下是每个阶段的详细说明：

\begin{enumerate}
\item 离线领域适配阶段：在这个阶段需要收集和整理用于模型微调的指令数据。本文所使用的训练数据主要来源于上一章节基于自建军事新闻语料构建的银标数据集，并将其进一步组织为适用于指令微调的训练样本。在此基础上，选取开源大语言模型 DeepSeek-R1-Distill-Qwen-32B 作为基座模型，并采用 QLoRA 训练策略进行参数高效微调。通过冻结原始参数、对模型权重进行低比特量化，并在部分线性层中引入低秩适配矩阵，可以在较低训练开销下完成领域知识抽取能力的适配，使得方法适用于各种显卡算力及显存容量有限的环境，大大拓展了本章方法的应用范围。
\item 候选结果生成：在完成领域适配后，将新闻段落输入到知识抽取模型中，由模型根据任务指令生成结构化的抽取结果。与单次确定性生成方式不同，本文针对同一输入文本采用受控随机采样策略生成若干个候选抽取结果集合。每个候选结果集合均对应一次完整的实体关系联合抽取结果，其中既包含实体之间的关系信息，也包含用于后续结构化组织的属性信息。通过生成多个候选结果，可以为后续结果筛选提供更充分的比较空间。
\item 本体约束与候选评分：在候选结果生成后，需要结合领域本体规则对抽取结果进行进一步分析。本文所使用的约束信息主要来源于的领域本体模型，包括关系类型集合、实体类型集合以及实体和关系之间的匹配规则。对于每个候选结果，分别从模型置信度、本体规则满足度以及错误项数量三个方面进行综合评分。其中，模型置信度由生成序列的 logprob 给出，规则满足率用于衡量候选结果中关系与本体约束的一致程度，错误项则主要包括重复三元组、格式错误、实体边界异常以及关系越界等问题。经过归一化处理后，上述指标共同构成候选结果的综合得分。
\item 结构化输出：在完成候选评分之后，选取得分最高的候选结果作为最终输出结果，并以统一的结构化 JSON 形式返回。通过这一过程，模型输出结果不再仅依赖单次生成，而是在领域适配、本体规则约束与候选结果选择的共同作用下得到进一步校正，这样不仅可以提升抽取结果的稳定性和结构合法性，也能够减少不合理三元组和错误关系的产生。
\end{enumerate}

\subsection{基于 QLoRA 的领域适配}

为了使 Student 端模型具备军事新闻领域的实体关系联合抽取能力，本文首先对开源大语言模型进行领域适配。其核心思路是：基于上一章构建的银标数据，组织面向军事新闻知识抽取任务的指令微调数据集，再结合参数高效微调技术 QLoRA，对基座模型进行低成本的任务适配，使其能够理解军事新闻文本中的领域表达，并按照指令输出结构化的抽取结果。整体流程与上一章银标数据构建结果相衔接，也为后续的本体约束过滤与候选结果重排序提供了基础模型支撑。

\subsubsection*{（1）指令数据的构建}

构建高质量指令数据是实现领域知识抽取的基础。结合本文的研究目标，Student 端需要完成的是面向军事新闻段落的实体关系联合抽取任务。因此，在领域适配阶段，训练数据不仅需要覆盖军事新闻中的主要实体类型和关系类型，还需要尽量保证输出结果具有统一的结构化形式，便于后续进行规则过滤、候选比较以及图谱入库处理。

本文的训练数据主要来源于上一章方法在自建军事新闻语料库上构建的银标数据。相较于直接使用通用领域信息抽取数据，这部分数据在文本来源、任务形式和输出目标上与本文的研究场景更加一致，能够较好反映军事新闻中的高频实体、关系表达及其组合模式。因此，本文以该部分银标数据为核心，对其进一步整理，构建面向 Student 端指令微调的训练样本。

在具体构建过程中，本文以新闻段落作为基本输入粒度。每条训练样本主要由三个部分组成：

1）\textbf{任务指令}：用于明确当前任务是军事新闻文本的实体关系联合抽取任务，并规定输出需要满足结构化结果的组织要求；

2）\textbf{输入文本}：即待抽取的军事新闻段落文本，作为模型进行语义理解、实体识别和关系判断的主要依据；

3）\textbf{目标输出}：即由银标数据转换得到的结构化抽取结果，用于指导模型学习从新闻段落到实体关系结果的映射过程。

通过这种方式，可以将上一章已经构建完成的银标数据自然衔接到 Student 端训练流程中，使模型在训练阶段逐步学习军事新闻文本中的术语表达、叙事模式以及结构化输出要求之间的对应关系。

需要指出的是，本文的抽取对象是实体关系联合抽取，而不是将实体抽取和关系抽取拆分为两个彼此独立的阶段。也就是说，给定一段军事新闻文本，模型需要在单次生成过程中同时完成实体识别、关系判定以及结果组织。输出结果采用统一的 JSON 三元组结构表示，其中实体属性信息也包含在相应结构化结果中，但本小节不展开具体 JSON 模板。这样设计的目的，是尽量减少流水线方式中误差逐级传播的问题，同时提高输出结果在结构层面的完整性与一致性。

\subsubsection*{（2）基座模型选择}

在基座模型选择方面，本文采用 DeepSeek-R1-Distill-Qwen-32B 作为 Student 端领域适配的基础模型。选择该模型主要有以下几个方面的考虑。

首先，该模型属于可获得的开源大语言模型，具备较好的中文文本理解与生成能力，能够为后续的领域任务适配提供稳定的语言基础。军事新闻文本包含较多专有名词、复杂事件描述以及关系性表达，这类文本对模型的语义理解能力和指令跟随能力都有较高要求，因此需要选取具有较强通用语言能力的基座模型作为出发点。

其次，相较于闭源在线模型，开源模型更便于在本地环境下进行训练、推理和部署，这与本文面向轻量化本地部署的研究目标是一致的。本文关注的不只是模型在实验环境中的抽取能力，还包括其在资源受限条件下的可训练性、可部署性以及后续持续运行的可行性。因此，在基座模型选择上，需要兼顾模型能力和工程落地成本。

最后，DeepSeek-R1-Distill-Qwen-32B 在保留较强通用语言能力的同时，也为后续通过参数高效微调完成任务迁移提供了较好的基础。对于本文所研究的军事新闻知识抽取任务而言，完全重新训练一个领域模型成本过高，而直接使用通用模型又难以稳定输出符合任务要求的结构化结果。因此，基于现有开源基座模型进行领域适配，是更符合本文研究目标的技术路线。

\subsubsection*{（3）QLoRA 训练策略}

在模型训练策略方面，本文采用 QLoRA 方法完成基座模型的领域适配。之所以不采用全参数微调，是因为对于参数规模较大的大语言模型而言，全参数微调的显存占用和训练成本都较高，不利于在资源有限的环境下开展训练，也与本文强调的轻量化部署目标不完全一致。相比之下，QLoRA 通过量化和低秩适配的方式，在尽量保持模型性能的同时，显著降低了训练所需的参数量和显存占用。黄俊涛在其第四章中也正是沿着“指令数据集 + 开源基座模型 + QLoRA 微调”的路径来构建科技领域实体关系抽取模型，这一写法对本节具有直接参考价值。 

QLoRA 的基本思想可以概括为以下三个方面：

1）\textbf{冻结原始参数}：在训练过程中，不直接更新基座模型的大部分预训练参数，而是将这些参数作为稳定的语言知识底座保留下来，从而避免全参数优化所带来的高计算代价；

2）\textbf{低比特量化}：将基座模型权重以低比特形式进行压缩表示，以减少显存占用，降低较大参数规模模型的训练门槛；

3）\textbf{低秩适配矩阵}：在模型的部分线性层中引入可训练的低秩矩阵，用于学习下游任务所需的参数增量，使模型能够在不改变原始大规模参数的前提下完成对军事新闻抽取任务的适配。

从训练流程上看，本文首先对基座模型进行量化处理，并在指定层中插入可训练的低秩适配矩阵；然后，基于前文构建好的军事新闻指令微调样本，对这些新增参数进行监督训练，使模型逐步学习军事新闻领域中的实体边界、关系触发模式以及结构化输出方式；最后，得到一个能够根据新闻段落输入生成结构化抽取结果的 Student 端基础模型。

这里需要强调的是，本节的领域适配主要解决的是“模型能否抽取”和“模型能否较稳定地生成结构化结果”这两个基础问题。也就是说，经过 QLoRA 微调后，模型已经能够在军事新闻文本上完成实体关系联合抽取，并按任务要求输出统一的 JSON 三元组结果。但对于输出结果是否满足领域本体约束、是否存在重复三元组、是否存在关系越界以及多个候选结果之间如何进行优选等问题，还需要在后续小节中进一步处理。因此，QLoRA 领域适配并不是本章全部“可控性”的来源，而是后续本体约束与候选评分机制得以发挥作用的前提。

\subsection{领域本体模型约束}

为了使 Student 端输出结果能够满足知识图谱构建的结构要求，本文在模型生成之后引入领域本体模型约束，对候选抽取结果进行合法性检查与结果过滤。这里所使用的约束规则，不再重新定义新的实体和关系集合，而是直接采用第三章已经构建完成的军事新闻领域本体模型，特别是表 3.2 所定义的实体类型集合和表 3.3 所定义的关系类型集合，并据此建立实体类型—关系类型—实体类型的匹配规则。这样做的目的，是将模型输出限制在已有 schema 所允许的结构空间内，减少关系越界、类型不匹配以及结果难以入库等问题。

\subsubsection*{（1）约束对象}

本文的本体约束对象，是 Student 端对单个新闻段落生成的结构化候选结果集合。每个候选结果中包含若干实体、关系及其对应的结构化字段。对于这些结果，系统重点检查以下三个方面：

1）\textbf{实体类型是否合法}：判断候选结果中的实体类型是否属于军事新闻领域本体预先定义的实体类型集合，例如 MilitaryNewsEvent、Actor、MilitaryAsset、Location、Time、QuantityValue、TaskContent、EventResult 和 MaterialResource 等；

2）\textbf{关系类型是否合法}：判断候选结果中的关系标签是否属于本体模型预定义的关系集合，例如 hasActor、hasMilitaryAsset、hasLocation、hasTime、hasQuantity、hasTaskContent、hasEventResult、hasRecipient、hasDeliveredAsset 等；

3）\textbf{关系连接是否合法}：判断一条关系两端的实体类型是否满足领域本体所规定的连接方式，即是否符合“头实体类型—关系类型—尾实体类型”的匹配要求。

其中，前两项约束解决的是“标签是否越界”的问题，第三项约束解决的是“关系能否成立”的问题。对于知识图谱构建来说，后者更为关键，因为即使关系名称本身属于预定义集合，若头尾实体类型不匹配，该结果仍然不能直接作为合法知识写入图谱。

\subsubsection*{（2）匹配规则构建}

结合第三章的本体设计可以看出，军事新闻知识表示具有明显的事件中心特征，多数关系都以军事新闻事件实体为核心，再连接到行动主体、军事资产、地点、时间、数量和事件结果等不同类型的尾实体。因此，本文将本体约束进一步整理为显式的类型匹配规则集合，用于判断候选三元组是否满足 schema 要求。

例如，hasActor 的尾实体应当属于 Actor 类型，hasLocation 的尾实体应当属于 Location 类型，hasTime 的尾实体应当属于 Time 类型，hasQuantity 的尾实体应当属于 QuantityValue 类型，hasEventResult 的尾实体应当属于 EventResult 类型。类似地，hasDeliveredAsset、hasTargetAsset、hasCarrierAsset 等关系，也应分别连接到对应的 MilitaryAsset 类型实体。通过这种方式，原本停留在概念层面的本体模型，被转化为可以直接执行的规则集合。

为了统一表示，可将领域本体中允许的合法连接关系记为
\[
\Omega=\{(T_h,r,T_t)\},
\]
其中，$T_h$ 表示头实体类型，$r$ 表示关系类型，$T_t$ 表示尾实体类型。对于任意候选三元组 $(e_h,r,e_t)$，若满足
\[
(type(e_h),r,type(e_t))\in\Omega,
\]
则认为该三元组在结构上满足本体约束；否则，将其视为不合法结果。这个判断过程不依赖人工逐条筛查，而是通过预先整理好的类型匹配规则自动完成。

\subsubsection*{（3）生成后的约束方式}

本文中的本体约束并不是在模型生成阶段以强制解码方式直接介入，而是在候选结果生成之后再执行规则检查。也就是说，Student 端首先根据输入新闻段落生成结构化候选结果，然后系统再依据领域本体规则对这些结果进行逐条检查和处理。这样设计主要是为了保持生成过程的完整性，同时避免在前向生成阶段过早施加过强限制，影响模型对复杂新闻表达的正常抽取。

在生成后的约束处理中，系统主要执行以下几个步骤：

1）\textbf{关系集合检查}：判断候选结果中的关系标签是否属于军事新闻领域本体规定的关系集合。若不属于，则认定为关系越界结果；

2）\textbf{类型匹配检查}：根据预定义的匹配规则，检查关系两端实体类型是否满足合法连接要求。若头尾实体与关系类型之间不匹配，则认定为非法三元组；

3）\textbf{结构完整性检查}：判断候选结果是否满足统一结构化输出的基本要求，避免因字段缺失、结构残缺而影响后续处理；

4）\textbf{异常结果检查}：识别重复三元组、实体边界异常等明显影响结果质量的问题，为后续结果筛选提供依据。

经过上述处理后，明显违反本体约束的结果将被过滤，或在后续候选选择阶段作为负向因素处理；满足本体约束的结果则被保留下来，进入下一步候选比较流程。这样一来，本体约束所承担的任务就比较明确：它不负责重新生成结果，而是负责对模型输出进行结构合法性校验，使候选结果更加符合领域 schema 的要求。

\subsubsection*{（4）约束作用}

本体约束在本文中的作用，主要体现在两个方面。

1）\textbf{保证结果结构合法}：通过显式的类型匹配规则，将模型输出限制在领域本体允许的关系空间内，减少非法关系和不合理连接；

2）\textbf{提升结果可用性}：经过本体约束处理后的候选结果，在结构一致性、结果规范性和入库可行性方面更稳定，便于后续进行候选排序、知识组织和图谱写入。

需要说明的是，本节的本体约束并不直接决定最终输出结果，而是为后续候选结果的选择提供基础。换言之，经过本体规则检查之后，系统已经能够区分“结构上可接受的候选结果”和“明显不合法的候选结果”，而最终从多个候选中选择哪一个，则在下一小节结合模型置信度、规则满足度和错误惩罚项进一步完成。

综上，本文在 Student 端引入的领域本体模型约束，主要任务是将第三章已有的军事新闻领域本体转化为可执行的结果检查规则，并在模型生成之后对候选结果进行实体类型、关系类型和关系连接方式的合法性校验。通过这一处理，可以有效减少关系越界和类型不匹配问题，为后续多候选结果的筛选与稳定入库提供结构基础。

\subsection{多候选结果生成与评分}

（这里需要解释候选结果生成策略、综合评分方法、最优结果输出机制）

为降低单次生成结果的随机波动，本文对同一输入新闻段落不只生成一次抽取结果，而是采用受控随机采样方式生成多个候选结果集合，并进一步结合模型置信度、本体规则满足度和错误惩罚项进行综合评分，选取得分最高的候选作为最终输出。这样做的目的，是在保持生成多样性的同时，尽量选出结构更稳定、规则更一致、错误更少的抽取结果。

\subsubsection*{（1）多候选结果生成}

对于每一段输入文本，本文使用经过领域适配后的 Student 端模型进行多次独立解码，得到 $K$ 个候选抽取结果，记为
\[
\mathcal{Y}=\{Y_1,Y_2,\dots,Y_K\},
\]
其中每个 $Y_k$ 表示该输入对应的一个完整抽取结果集合，而不是单条三元组。每个候选结果内部可以包含若干实体、关系以及相应属性信息，并统一组织为结构化 JSON 三元组形式。

在候选生成阶段，本文采用 top-$p$ 采样（nucleus sampling）作为解码策略。其基本思想是：在生成每一个 token 时，不直接在全部词表上进行随机采样，而是先按照概率从高到低对候选 token 排序，再选取累计概率达到阈值 $p$ 的最小 token 集合，最后只在这一集合中进行采样。这样可以避免从概率过低的 token 中随意采样，同时保留一定的生成灵活性。相较于贪心解码，top-$p$ 采样能够得到更加多样的候选结果；相较于完全无约束的随机采样，它又能将生成过程控制在较高概率区域内，更适合本文的结构化抽取任务。

若将当前时刻的候选 token 概率分布记为 $\{q_1,q_2,\dots\}$，并按概率从高到低排序，则 top-$p$ 采样对应的候选集合可写为
\[
V_p=\left\{x_1,x_2,\dots,x_m \,\middle|\, \sum_{i=1}^{m} q_i \ge p \right\},
\]
其中 $V_p$ 表示当前步用于采样的 nucleus 集合。模型仅在该集合内进行下一 token 的生成。

在本文中，一个候选结果的比较单位是“一次完整输出”，而不是其中的某个局部关系。这是因为实体关系联合抽取结果具有整体结构特征，同一候选内部的实体识别、关系判断和属性组织通常相互关联。若只对单个三元组独立处理，容易破坏整体结果的一致性；而以完整结果集合作为比较对象，更符合后续结构化解析和图谱入库的处理方式。

\subsubsection*{（2）模型置信度计算}

得到多个候选结果之后，需要首先衡量每个候选结果本身的生成可信程度。本文采用生成序列的 logprob 作为候选结果的基础置信度信号。

这里的 logprob 指的是模型在生成某个输出序列时，对该序列中各个 token 所赋予概率的对数值之和。设候选结果 $Y_k$ 对应的输出序列为 $(y_1,y_2,\dots,y_n)$，则其序列 logprob 可表示为
\[
\log P(Y_k)=\sum_{t=1}^{n}\log P(y_t \mid y_{<t},x),
\]
其中 $x$ 表示输入新闻段落，$y_{<t}$ 表示当前位置之前已经生成的 token。直观地说，若某个候选结果的 logprob 较高，说明模型在当前输入条件下更倾向于生成该结果。

由于不同候选结果的序列长度可能不同，若直接比较总 logprob，长序列往往会因为累积项更多而天然偏低。因此，本文对 logprob 做长度归一化处理，将其转化为候选结果的置信度分量 $Conf_k$。一种简单做法是采用平均 token 对数概率，即
\[
Conf_k=\frac{1}{n}\sum_{t=1}^{n}\log P(y_t \mid y_{<t},x).
\]
通过这种方式，可以在不同长度的候选结果之间进行相对公平的比较。

\subsubsection*{（3）规则满足度与错误惩罚项}

仅依靠模型置信度并不足以选出最适合入库的结果，因此还需要进一步考虑候选结果与领域本体规则的一致性，以及结果中存在的结构性错误。

首先是\textbf{规则满足度}。设候选结果 $Y_k$ 中共包含 $N_k$ 个关系三元组，其中满足本体约束规则的三元组数量为 $M_k$，则其规则满足率定义为
\[
Rule_k=\frac{M_k}{N_k}.
\]
其中，本体约束规则主要依据第三章中已经定义好的实体类型集合、关系类型集合以及实体类型—关系类型—实体类型匹配规则进行检查。若某个候选结果中的关系大部分满足领域 schema 要求，则其 $Rule_k$ 值较高；反之，若其中存在较多关系越界或类型不匹配情况，则该项得分较低。

其次是\textbf{错误惩罚项}。在本文中，错误项主要包括以下几类：

1）\textbf{重复三元组}：同一候选结果中出现完全重复或语义重复的关系三元组；

2）\textbf{格式错误}：输出未能满足统一 JSON 结构要求，导致后续解析困难；

3）\textbf{实体边界异常}：实体片段明显截断、冗长，或与原文表达不对应；

4）\textbf{关系越界}：关系类型超出预定义关系集合，或头尾实体类型与关系不匹配。

设候选结果 $Y_k$ 对应的错误惩罚值为 $Error_k$。该值越大，说明候选结果中存在的问题越多，在最终评分中应受到更大惩罚。

\subsubsection*{（4）综合评分与最优结果选择}

为统一比较多个候选结果，本文从模型置信度、本体规则满足度和错误惩罚项三个方面构建综合评分函数。由于 $Conf_k$、$Rule_k$ 和 $Error_k$ 原本不在同一量纲区间，本文在加权组合前先对三者进行归一化处理，分别记为 $\widetilde{Conf}_k$、$\widetilde{Rule}_k$ 和 $\widetilde{Error}_k$。

候选结果 $Y_k$ 的综合得分定义为
\[
Score_k=\alpha \cdot \widetilde{Conf}_k+\beta \cdot \widetilde{Rule}_k-\gamma \cdot \widetilde{Error}_k,
\]
其中，$\alpha$、$\beta$ 和 $\gamma$ 分别表示模型置信度、本体规则满足度和错误惩罚项的权重系数。该公式对应的含义比较直接：候选结果的生成概率越高、越符合本体规则、包含的错误越少，则其最终得分越高。

在此基础上，对同一输入文本生成的 $K$ 个候选结果进行统一评分，并选取得分最高的候选作为最终输出。设评分集合为
\[
\{Score_1,Score_2,\dots,Score_K\},
\]
则最终选择结果为
\[
Y^{*}=Y_{k^{*}},\quad k^{*}=\arg\max_k Score_k.
\]

这样，最终输出不再依赖某一次单独生成，而是在多个候选结果之间经过显式比较之后产生。该过程能够在一定程度上缓解单次解码路径带来的偶然性，使系统更倾向于保留“置信度较高、规则更符合、错误更少”的结果。
